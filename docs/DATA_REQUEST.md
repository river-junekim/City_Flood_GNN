# 데이터 요청서 — 한국건설기술연구원(KICT) 앞

> 도심 침수 예측 프로젝트 (원 목표: 관악구 배수지역 침수 예측)
> 작성 2026-06-24 · 현재 분석 현황은 `reports/progress_report.html`(§1~18)·`docs/BRAINSTORM_2026-06-24.md` 참조

## 배경 (요청 사유 요약)
현재 확보한 **서울 도로노면(123)·하수관로(484) 수위센서 + AWS 강우 + 관악 GIS + 도림천 수위**로 분석한 결과, 반복 확인된 병목은 모델이 아니라 **데이터**입니다.
- 도로 침수 라벨의 ~93%가 센서 아티팩트(stuck·단일값) → 정답으로 부적합
- 관악 침수는 하수 만관도, 도림천 하천역류도 아님(검증 완료) → **국지 호우 표면류**로 좁혀짐
- 위험수위(충전율≥0.8) 예측은 가능하나 국지강우만으론 리드타임이 없어 persistence를 못 넘음

→ 아래 데이터가 확보되면 **실제 침수 라벨 기반 supervised 모델 + 조기경보**가 가능해집니다. 레이더 강우(HSR)는 기상청 API로 별도 수신 예정이라 제외했습니다.

## A. KICT 자체 연구자산 (직접 제공 요청)
| 항목 | 왜 필요한가 |
|---|---|
| **고해상 DEM / LiDAR (1m급)** | 관악 저지대 표면류 위험지도·도로센서 그래프 구성. 공개 DEM보다 정밀 필요. |
| **하수관망 수리모델(SWMM 등)·배수해석 결과** | 관별 통수능력·흐름방향(현 GIS 보강). 위험수위→월류 물리해석. |
| **레이더 강우 보정/합성 강수장** | KMA 원자료보다 보정된 격자. (내일 API 수신과 별개로 품질 향상) |
| **침수 실측·검증 데이터(침수심·흔적·현장조사·CCTV)** | ★ 모델의 정답 라벨. KICT 연구 검증에 쓰는 데이터가 곧 우리 ground-truth. |

## B. 서울시 데이터 중개 요청 (우리 직접 확보 곤란)
| 항목 | 왜 필요한가 |
|---|---|
| **수위센서 정비·교정·고장 이력** | ★ 도로 라벨 93% 아티팩트의 원인 규명 → 어떤 센서가 왜 stuck인지 알면 일부 라벨 **구제** 가능. |
| **침수흔적도 상세본 + 120/119 침수 신고 기록** | 실제 침수 시각·지점·심도 = 다년 정답 라벨. |
| **2022년 8월 신림 침수 상세 기록** | 다년 사건(현 도로센서는 2024~만, 하수는 2022~ 보유하나 라벨 없음). |
| **빗물받이 막힘·준설·청소 이력** | 관악 표면류 침수의 직접 원인(위치는 GIS 보유, 상태/이력은 미보유). |
| **빗물펌프장·수문·저류조 운영 이력** | 비강우 위험수위(만관)의 운영적 원인 설명. |

## C. 데이터 외 — 방향 정렬을 위한 확인 (꼭 회신 요청)
1. **연구 목표 정의**: "AI 침수 예측 모델"인가, "위험 조기경보/디지털트윈"인가? → 납품물·평가지표(CSI vs 리드타임/POD)가 갈립니다.
2. **KICT의 ground-truth 정의**: 침수 검증 시 무엇을 정답으로 쓰는지. 이미 검증 데이터가 있다면 라벨 병목이 즉시 해소됩니다.
3. **센서 부지 선정 근거**: 상습침수 기반인지 등(센서 위치 해석에 필요).

## 우선순위 (받는 순서 추천)
1. **고해상 DEM + 하수관망 수리모델** (KICT 자체, 바로 제공 가능)
2. **침수 실측/검증 데이터 = KICT ground-truth** (라벨 병목 직접 해소)
3. **센서 정비·고장 이력** (아티팩트 라벨 구제)
4. 침수흔적도·120/119, 2022 신림 사건, 빗물받이 막힘 이력

> 위 1~3만으로도 현재 최대 병목(라벨·표면류 원인·센서 품질)이 크게 풀립니다.

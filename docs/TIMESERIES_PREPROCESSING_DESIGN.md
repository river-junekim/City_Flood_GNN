> 작성일: 2026-06-17
> 목적: 일반 전처리가 아닌 **시계열 데이터 전처리 원칙**으로 도로노면·하수관로 센서를 재정의한다.
> 강우(공공데이터)는 제외하고 두 데이터만으로 시작, 추후 강우(AWS) 추가 예정.

## 0. 배경 — 왜 재설계하나 (EDA 근거)

- `flood_flag = level > 0` 폐기: 도로 "1cm"는 대부분 baseline 노이즈(연중 8.3%, 마른 1월에도 56%).
- 도로는 **이산 단계 게이지**(0,1,2,3,4,6,11,...,96). `96`=측정범위 천장 saturation 오류.
- 하수는 **연속 기저유량**(95% >0), surcharge는 드묾.
- 변환·처리는 정상(원본=parquet 검증). 문제는 **라벨 정의 + 센서 아티팩트**.
- 하수→도로 선행성은 **미확정**(과거 분석은 오염 라벨·거리매칭 기반) → 단정 금지, 검증으로 판단.

## 1. 시계열 전처리 원칙 (일반과 다른 점)

| 단계 | 일반 전처리 | 시계열 전처리(우리) |
|------|------------|-------------------|
| 결측 | 평균/0 대체 | **마스크 지시자 + 갭길이별**, 보간 최소 |
| 이상치 | 값 범위만 | **stuck-run(frozen)·스파이크** 시간축 탐지 |
| 정상화 | 스케일만 | **차분·센서별 baseline 제거** |
| 피처 | 임의 | **causal rolling/lag(과거만)** |
| 분할 | 무작위 셔플 | **시간순 + 경계 갭**(누수 방지) |
| 평가 | accuracy | **persistence 베이스라인 능가** 필수 |

## 2. 확정된 결정 (lock-in)

```text
과제      : nowcast — 타겟 t+H, H=1(다음 10분)로 시작
            → H를 키우면 forecast로 확장 (설계상 동일 파이프라인)
순환방지  : 타겟 시점(t+H)을 입력 윈도우에 미포함 (자기값으로 자기 맞히기 금지)
해상도    : 10분 격자
결측      : 마스크 중심 (is_missing 지시자, 긴 갭 masking, 보간 최소)
입력윈도우: T_in = 6 스텝(60분, 과거만) — 추후 3/6/12 튜닝
집계      : 도로 = max + 지속시간(10분 중 ≥T인 분 수)
            하수 = mean/max/min/last
차분      : level_diff를 '피처'로 추가 (도로·하수) — 타겟/유일표현 아님
평가      : persistence(직전값) 베이스라인을 반드시 능가
패널형태  : (sensor_id, ts10) + causal 피처 + 정렬 라벨 (강우 조인·H시프트 대비)
```

### 2-1. 과제 — nowcast vs forecast
- 강우 없이 긴 horizon forecast는 비현실적 → **nowcast(H=1)로 골격**, 강우+H로 forecast 확장.
- 구조: `타겟 = label.shift(-H)`, 입력은 과거 causal 피처. H만 바꾸면 전환.

### 2-2. 차분(Differencing)
- **피처로 추가**: 변화율(상승속도) = 도로 onset / 하수 surcharge 신호. 센서별 baseline 자동 제거(정상성).
- **타겟·유일표현 X**: 침수 탐지는 '수위(물 있나)'가 신호 → level과 level_diff를 **함께**.
- 주의: ① **정제 후 차분**(노이즈 증폭 방지) ② **갭 너머 차분 금지**(마스크 존중) ③ 10분 격자 ④ 상승분 `max(diff,0)` 분리 고려.

## 3. 도메인 분리 설계

각 도메인을 **1급으로 독립 정제**하고, 조인은 분리된 레이어로 미룬다(관계 판정은 데이터가 하게).

```text
[도로 도메인]  road_cleaned + road_sensor_quality + road_panel_10min
[하수 도메인]  sewer_cleaned + sewer_sensor_quality + sewer_panel_10min(+fill_rate)
[조인 레이어]  road↔sewer 페어링(수리적 우선/거리 차선) — deliberate, 나중에
```

- 둘 다 **(sensor_id, ts10) 키 + 품질플래그 + causal 피처**로 통일 → 조인 용이.
- 하수 보강 필요: **품질등급**(stuck 하수센서 검출) + **surcharge 피처**(fill_rate=level/관용량, 천장근접도, level_diff).
- 도로는 대부분 완료(road_cleaned, road_sensor_quality 존재). 단 시계열 원칙(마스크·차분·지속시간)으로 재정비.

## 4. 이상치 처리

### 4-1. 하드 제거 (값 기반)
- 도로 에러코드 `{312,419,999,1000}` → NaN.
- 도로 `96` = 측정범위 천장 saturation → NaN (단, 센서별 측정범위 기준이 더 견고).
- 하수 음수/측정범위 초과 → NaN.

### 4-2. stuck-run 검출 (시간축, frozen 센서)
- **정의**: 비영점 동일값이 길게 연속 = 센서 frozen.
- **0은 제외**: 도로 "0 연속"=마름(정상), stuck 아님.
- **두 단계**: 센서-단위(전체 품질등급) + **구간-단위**(특정 기간만 frozen → 그 구간 마스킹).
- **도메인 차이**:
  - 하수(연속): 정확 동일값 연속 = 비교적 명확.
  - 도로(이산): 진짜 침수도 한 단계에 머뭄 → **보수적**, run-length 단독 판정 위험.

#### run-length 분포 분석 결과 (2026-06-17)
비영점 동일값 연속 run의 시간 길이 측정:

| | 시간가중 분포 핵심 | ≥6h run |
|---|---|---|
| **도로** | **1d+ 가 97.5%** (비영점 시간 거의 전부가 하루 이상 연속) | 1,142개 / 80센서. "1"이 ~31일(한 달) 연속 |
| **하수** | 5분~6h에 각 12~15% 골고루 + **1d+ 17.4%** | 54,151개 / 441센서. base값이 한 달 고정 |

**해석:**
- 도로 비영점은 대부분 month-long **stuck-at-1**(baseline) — 단 라벨에선 T≥6으로 제외되는 값.
- 하수는 30분~6h run에 시간이 퍼짐 = **잔잔한 기저유량도 한두 시간 같은 값** 가능 → **60분 임계는 정상 오탐**.
- **명확한 stuck = "하루 이상 연속"** 이 양 도메인 공통 신호.

#### 임계 — 추측 폐기, 데이터 기반 수정
| 도메인 | (초기 추측) | **데이터 기반** |
|--------|------------|----------------|
| 하수 | ~~60분~~ | **≥1day (또는 ≥12h)** — 안정 기저유량 오탐 회피 |
| 도로 | ~~6시간~~ | **≥1day** (1d+가 압도) |

- ※ 위 분석은 baseline 값 포함이라, **라벨값(도로 level≥6 / 하수 고수위·surcharge) 한정 재분석**으로 정밀화 필요.
- ※ 도로 stuck의 진짜 해결은 **강우 추가 후 대조**("긴 비영점 run인데 비 없음=stuck").

## 5. 라벨 정의 (침수)

```text
침수 = (도로 level ≥ T) AND (오류/saturation 제외) [+ 강우는 추후]
```
- **T는 고객 운영기준** 확정 필요. 이산 게이지라 **T=6cm 유력**, 비교용 T=2/10도 생성.
- 도로 10분 max(+지속시간)으로 라벨 산출.
- 하수 `level>0`은 기저유량 → 라벨로 사용 금지.

## 6. 분할·스케일링·불균형

- **분할**: 시간순 train(2024)/val(2025 H1)/test(2025 여름) + **윈도우 경계 누수 방지**.
- **스케일링**: 정규화 파라미터 **train에서만 fit** → val/test 적용.
- **불균형**(진짜 침수 극희소): 점단위 무작위 오버샘플 ❌ → **이벤트/윈도우 단위 샘플링**, focal loss. 평가는 PR-AUC·event recall.

## 8. 처리 순서 (제안)

```text
1. 도로 도메인 재정비  (마스크·차분·지속시간·stuck 구간플래그)
2. 하수 도메인 1급화   (품질등급 + fill_rate/surcharge/diff + stuck)
3. 각 도메인 10분 패널 (join-ready, causal 피처, is_missing)
4. 라벨 생성 (T=2/6/10, 오류/saturation 제외)
5. 베이스라인(persistence 대비) — nowcast H=1
6. (추후) 강우 조인 → 라벨 recall·forecast·관계검증
7. (추후) 조인 레이어 + 그래프(고객 요구)
```

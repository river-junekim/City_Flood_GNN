# 2차 보고서 — 검증된 발견: 하수 다중센서가 실제 침수를 포착한다

> 기준: 관악구 · 작성 2026-07-02 · 1차 보고서(노드-링크 상관관계 분석)의 후속 · HTML: `reports/report_2nd_findings.html`
> 성격: 재조사로 **1차의 "현 데이터로 침수 양성 부재" 결론이 부분적으로 뒤집힘** — 하수 다중센서가 실제 침수 사건을 포착하고 있음이 검증됨. 그 검증된 사실을 정리한다.
> ⚠️ 모든 항목은 **관측·검증된 사실**과 **해석**을 구분해 기술. 강우 동반은 gate가 아니라 교차검증으로만 사용.

---

## 0. 핵심 전환 한 줄
1차에서 "관악 도로센서 신뢰 침수 양성 ≈ 0"으로 보았으나, **재조사 결과 하수센서가 실제 침수 사건을 다수 포착**하고 있었다. "양성 0"의 원인은 **센서 결함이 아니라 관측창 한계**(도로센서 2024-06 설치·AWS 강우 여름만 보유)였다.

## 1. 침수 판단 기준 통일 (검증된 결정)
- **도로 침수 = 도로수위계 침수심 ≥ 15cm** (출처: 서울시 침수예보 발령 기준, news.seoul.go.kr/env). 기존 임의 임계(≥6cm) 폐기.
- **하수 충전율(fill_rate, 만관/위험수위)은 침수 기준이 아니라 별개 신호로 분리.** (도로 표면 침수 ≠ 하수관 만관)

## 2. 2022-08-08 (알려진 관악 대침수일) — 하수 7센서 동시 포착 ★
당시 운영 중이던 관악 하수센서 **7개 전부**가 그날 저녁 동시 반응:

| 센서 | 지점 | peak fill | fill≥0.6 지속 | 피크시각 |
|---|---|---|---|---|
| 21-0001 | 삼성808 | 0.88 | 160분 | 21:00 |
| 21-0004 | 낙성대 | 0.82 | 150분 | 22:10 |
| 21-0007 | 신림1450 | 0.78 | 190분 | 23:00 |
| 21-0005 | 보라매968 | 0.74 | 130분 | 21:50 |
| 21-0003 | 미성 | 0.74 | 150분 | 20:50 |
| 21-0006 | 행운 | 0.71 | 80분 | 21:00 |
| 21-0002 | 조원547 | 0.68 | 20분 | 21:57 |

> ※ 위 수치는 정제 데이터(10분 집계)로 재현한 값(그림 `req_2022_hydrograph.png`). 피크·형태는 1분 원자료와 동일, 지속시간만 집계 단위 차이로 소폭 상이.
> ※ **전처리 정정**: 재현 중 하수 센서 일부에서 고수위 중 순간 바닥값(0.01m) 급락→복귀하는 **센서 드롭아웃**을 발견(데이터셋 전체 4,356건·10,977 bins·0.021%·100/485 센서). 고수위 사이에 낀 바닥값 런만 선형보간으로 정정(`is_dropout` 플래그 추가, 노트북 `notebooks/01_preprocessing/sewer_dropout_clean.ipynb`). 원자료(1분)는 보존. 예: 21-0005 2022-08-08 21:00~21:30(2.24m→바닥값 42분→2.15m 복귀)이 정정됨. 위 표·그림은 정정 후 기준.

- **시계열 형태 = 상승→피크→하강 수문곡선** (예: 21-0001 19시 0.25 → 21시 0.88 → 23시 0.47). 스파이크·고착 아님.
- 7센서가 **같은 날 저녁(20:55~23:08)** 에, **수십~수백 분 지속**으로, 피크 시각을 조금씩 달리하며 동시 반응 = 광역 침수의 교과서적 신호.
- 강우: 우리 AWS 강우창(2024-06~) 밖이라 직접 확인 불가하나, **2022-08-08 시간당 141.5mm·일 381.5mm**(신대방동 관측, 외부 기록)로 뒷받침. ※2023~ 강우 추가 수집으로 보강 진행 중.

## 3. 강우 교차검증 — 확인 가능한 다중센서 사건은 전부 강우와 일치 ★
AWS 강우를 보유/추가수집해 확인 가능한 다중센서 사건 5개가 **모두 실제 강우(30~42mm)와 시각 일치**:

| 날짜 | 동시반응 센서 | 관악 최대 시간강우 | 검증 |
|---|---|---|---|
| 2025-05-16 | 5 | 36.0mm | ✅ (강우 15:40 급증 → 센서 15:54~16:07 반응) |
| 2025-08-30 | 3 | 41.5mm | ✅ |
| 2025-07-19 | 2 | 41.0mm | ✅ |
| 2025-07-17 | 3 | 31.5mm | ✅ |
| 2024-07-22 | 3 | 30.5mm | ✅ |

→ **강우를 확인할 수 있는 경우, 하수 다중센서 반응은 예외 없이 강우와 맞물린다.** (하수센서가 강우성 침수/만관을 일관되게 포착)

## 4. AWS 강우 커버리지 갭 발견·보완 (검증의 전제)
- 기존 AWS 강우 = **여름(6~9월) 2024·2025만** 다운로드돼 있었음. → 비여름·과거 사건의 "강우데이터없음"은 **무강우가 아니라 미수집**.
- 2025-05 보완 완료(→ 2025-05-16 사건 검증). **2023~ 과거 강우 추가 수집 진행 중**(용량 제한으로 다회 분할·재개, 2026-07-02 재개 실행 중).

## 5. 재해석 — "양성 0"의 진짜 원인 (관측창 한계)
- **도로센서는 2024-06 설치** → 2022·2023 대형 침수를 데이터에 담지 못함.
- **AWS 강우는 여름만 보유** → 비여름·과거 강우 검증 불가.
- 따라서 1차의 "현 데이터로 supervised 학습 불가"는 *센서가 못 쓴다*가 아니라 ***우리 분석 창이 큰 사건을 비껴갔다*** 가 정확한 진술.

## 6. 모델 학습 관점 의의 + 데이터 요청 연결
- **하수 다중센서 동시반응 = 침수 정답 라벨 후보** (지속·동시·수문곡선·강우동반으로 신뢰도 평가 가능).
- 단 **깨끗한 다년 라벨**에는 (a) 과거 강우(2022·2023, 수집 중), (b) 침수흔적도·120/119 신고가 필요 → **데이터 요청 ①(검증된 침수 라벨)과 직접 연결.**
- 관측 밀도: 2022 당시 7센서 → 현재 13센서(신설 포함) → 신설 29 확보 시 42센서. 사건 포착력 향상.
- **노드 라벨 생성 완료(2026-07-02)**: 침수흔적도 폴리곤을 GNN 맨홀 노드 13,272개에 공간조인 → 2022-08-08 노드별 침수 라벨(내부 1,229·≤50m 6,709·침수심·원인). 센서 42노드 전부 라벨(사건 7센서 중 6/7 50m내). 라벨 부재로 막혔던 하수 라우팅 GNN에 실제 노드 정답 확보. 노트북 `02_analysis/floodtrace_manhole_join.ipynb`, 산출 `dataset/processed/eda_based/gnn_manhole_flood_labels_2022.parquet`·`reports/figures_demo/floodtrace_manhole_labels_2022.png`.

## 7. 외부 자료 — 침수흔적도(ground-truth)·서울시 2026 대책 (참고)
**(1) 침수흔적도 = ground-truth 침수 라벨 — 확보·대조 완료(2026-07-02) ★**
- 서울 열린데이터광장 **OA-15636**(서울시 침수흔적도, 2010~2025 연도별·공공누리 1유형)에서 **2022년분 확보**(`03_GIS/침수흔적도_shp/2022_서울시_침수흔적도.zip`, 19,881 폴리곤). safemap.go.kr(2002~2023)·safecity.seoul.go.kr도 제공.
- 속성: 구·침수심(F_SHIM)·발생일자/시각·**원인(F_RSN_DTL: 하수역류·배수용량초과 등)**·주소·면적·폴리곤(EPSG:5179).
- **대조 결과**(노트북 `notebooks/02_analysis/flood_trace_crosscheck.ipynb`): 관악 2022-08-08 침수 폴리곤 **2,120개**(원인 "배수용량초과"=하수 만관 1,002개 포함). **동시반응 7센서 전부가 실제 침수 폴리곤 위/인접**(최근접 ≤102m·중앙값 0m, 300m내 39~201폴리곤) → **센서 신호 = 실제 침수 위치·날짜 일치, ground-truth 검증 완료**.
- **시간축 정합**(노트북 `floodtrace_timing_align.ipynb`): 흔적도 기재시각(시각기재 1,115폴리곤)이 **저녁 20~23시에 72% 집중** → 센서 피크시각(20:50~23:00)과 일치. **공간+시간 양면 검증**. 사건 카탈로그 E01을 "흔적도 검증(공간+시각)"으로 승격.
- → 데이터 요청 ①(검증된 침수 라벨)의 상당 부분을 공공데이터로 확보 가능. 산출: `dataset/processed/eda_based/floodtrace_sensor_crosscheck_2022.csv`·`03_GIS/derived/침수흔적도_2022_관악.gpkg`·`reports/figures_demo/floodtrace_2022_gwanak.png`.

**(2) 서울시 2026 풍수해 안전대책 = 본 접근의 타당성 공식 입증**
- 서울시 2026: **"AI 기반 침수 예측 — 과거 강우량 + 도로·하수관로 수위 데이터 학습"**(강남역 등 15개소). = **본 프로젝트 접근(강우+도로/하수 수위 → 침수예측)을 서울시가 공식 채택.** (관악은 우선대상 미포함 → 기여 여지)
- **관악구 = 반지하 밀집 → 소형 레이더 수위계 30개 추가**('25년 15→'26년 45). = 데이터 요청 ④(신설 센서)와 직결, 관측 노드 밀집.
- 새 긴급재난문자 기준(72mm/1h 또는 50mm/1h&90mm/3h) — 강우 기반, 본 분석의 도로수위계 15cm 기준과 보완.
- 출처: safemap.go.kr · safecity.seoul.go.kr(침수흔적도·2026 풍수해 안전대책).

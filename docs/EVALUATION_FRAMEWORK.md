## 1. 회귀 예측 성능 평가 지표

### 지표별 수식·의미·장단점

| 지표 | 수식 | 장점 | 단점 | 침수 예측 적합성 |
|------|------|------|------|----------------|
| **MAE** | $\frac{1}{n}\sum\|y_i - \hat{y}_i\|$ | 원 단위(cm) 직관적 해석, 이상치 강건 | 큰 오차·작은 오차 동일 페널티 | ✅ **주 지표** |
| **RMSE** | $\sqrt{\frac{1}{n}\sum(y_i-\hat{y}_i)^2}$ | 큰 오차에 민감 → 위험수위 오차 탐지 유리 | 이상치 한 개로 전체 값 왜곡 | ✅ **주 지표** |
| **MSE** | $\frac{1}{n}\sum(y_i-\hat{y}_i)^2$ | 손실함수로 적합, 미분 용이 | 단위가 cm²라 해석 어려움 | ⚪ 학습용 |
| **R²** | $1 - \frac{\sum(y_i-\hat{y}_i)^2}{\sum(y_i-\bar{y})^2}$ | 스케일 독립, 수위 패턴 설명력 측정 | 편향 모델도 높게 나올 수 있음 | ✅ 보조 |
| **MAPE** | $\frac{100}{n}\sum\|\frac{y_i-\hat{y}_i}{y_i}\|$ | 상대 오차 직관적 | 수위=0(무강우)일 때 분모 0 | ❌ **사용 지양** |
| **SMAPE** | $\frac{200}{n}\sum\frac{\|y_i-\hat{y}_i\|}{\|y_i\|+\|\hat{y}_i\|}$ | MAPE보다 0 근처 안정적 | 수위 0~1cm 구간에서 여전히 불안정 | ⚪ 제한적 사용 |

### RMSE/MAE 비율의 해석

```
RMSE / MAE ≈ 1.0  → 오차가 고르게 분포 (이상치 없음)
RMSE / MAE >> 1.0 → 소수의 극단 오차 존재 (위험수위 근처 집중 실패 가능성)
```

> **운영 기준 권고**: MAE ≤ 3cm (일상 모니터링), RMSE ≤ 6cm (위험 이벤트 허용 기준)

### 지표 역할 배분

```
◆ 주 지표 (논문·보고서 Table 필수)
  ├── MAE  : "평균 오차 X cm"로 운영 기준에 직결
  └── RMSE : 위험수위 부근 큰 오차에 민감, MAE와 함께 제시

◆ 보조 지표
  ├── R²      : 수위 상승·하강 패턴 설명력 (0.85 이상 목표)
  └── SMAPE   : 센서별 스케일 차이가 있을 때 비교에 활용

◆ 사용 지양
  └── MAPE    : 무강우 구간(road_level=0) 비율 95%인 환경에서 실용 불가
```

## 2. 시간 구조 기반 평가

### 2-1. Horizon별 평가 (Step-wise Metrics)

현재 모델(T_out=3)에서 예측 시점을 분리하여 평가해야 합니다.

```
예측 Horizon  : t+10분     t+20분     t+30분
회귀 지표     : MAE@10    MAE@20    MAE@30
               RMSE@10   RMSE@20   RMSE@30
분류 지표     : Recall@10  Recall@20  Recall@30
               F1@10      F1@20      F1@30

기대 패턴     : MAE@10 < MAE@20 < MAE@30 (horizon 증가 → 오차 증가)
운영 기준     : MAE@30 ≤ 5cm 이상이어야 30분 전 대피 경보에 활용 가능

⚠️ 주의: 전체 T_out 평균만 보면 t+10min(쉬운 예측)이 t+30min을 가리는
         "희석 효과"가 발생하므로 반드시 분리 측정 필요
```

### 2-2. Lookback Window 비교 실험 설계

| Lookback | 스텝 수 (10분 해상도) | 포착 가능한 패턴 | 권장 용도 |
|----------|---------------------|----------------|---------|
| 1시간 | 6 (현재) | 현재 수위 추세 | 빠른 베이스라인 |
| 3시간 | 18 | 강우 시작 ~ 수위 반응 지연 (5~15분) | 이벤트 탐지 |
| **6시간** | **36** | **집중호우 전체 사이클** | **권장** |
| 12시간 | 72 | 태풍 전면 강우 포착 | 계절성 이벤트 |

```
비교 방법:
  동일 아키텍처(DCRNN)를 lookback만 변경하여 4회 학습
  → 구간별(강우 시작·첨두·하강) MAE 분리 기록
  → 단기 이벤트: 짧은 lookback 유리 가능
     장기 지속 강우: 긴 lookback 유리
```

### 2-3. Time Step 단위 비교 (5분 / 10분 / 30분)

| Time Step | 시퀀스 길이 (동일 60분 입력) | 모델 관점 | 운영 관점 |
|-----------|--------------------------|---------|---------|
| 5분 | 12스텝 | 조밀, 계산 비용 3배, 노이즈 민감 | 빠른 반응, 허위경보 증가 |
| **10분** | **6스텝 (현재)** | **균형점** | **표준 관측 주기** |
| 30분 | 2스텝 | 단순화, 세밀 변화 손실 | 장기 예측에 적합, 단기 이벤트 탐지 불가 |

```
공정 비교 기준:
  동일 실제 예측 시간 (30분 후) 기준으로
  5분 step → 6스텝,  10분 → 3스텝,  30분 → 1스텝 예측
  → MAE@30min 동일 단위로 직접 비교
```

## 3. GNN 구조 기반 평가 (Node-wise)

### 3-1. Node-wise 평가가 필요한 이유

도로 침수는 **국지적 현상**입니다. 전체 평균 MAE가 낮아도 특정 노드 1~2개에서 오차가 집중될 수 있으며, 이 노드가 실제 침수 취약 지점일 경우 운영 실패로 이어집니다.

```python
# 노드별 MAE 계산 (shape: 배치×노드×horizon)
node_mae  = np.mean(np.abs(y_true - y_pred), axis=(0, 2))  # (N_road,)
node_rmse = np.sqrt(np.mean((y_true - y_pred)**2, axis=(0, 2)))

# 보고 지표
median_mae = np.median(node_mae)         # 대표 노드 성능
p90_mae    = np.percentile(node_mae, 90) # 최악 10% 노드 성능
```

### 3-2. 공간 성능 분석 방법

| 분석 | 방법 | 시각화 |
|------|------|------|
| 노드별 오차 분포 | 박스플롯 (90개 도로 노드) | 이상 노드 식별 |
| 배수구역별 성능 | `자치구` 기준 MAE 집계 | 히트맵 |
| 공간 편향 | Moran's I 통계 (오차 공간 자기상관) | ≈0이면 무작위 오차 |
| 엣지 밀도 vs 오차 | 노드 엣지 수 vs MAE 산점도 | 그래프 구조 영향 확인 |

### 3-3. 특정 노드 오차 집중 시 의심 원인

| 원인 유형 | 판별 방법 | 대응 |
|----------|---------|------|
| **Fallback 연결 노드** | `edge_type='fallback'`, 2km 거리 연결 | 가중치 감소 또는 제외 |
| **고립 노드** | sewer→road 엣지 수 = 1개 | 추가 엣지 구성 |
| **센서 품질 불량** | 해당 노드 결측률, 이상치 비율 | 전처리 강화 |
| **특수 지형** | 저지대·지하차도·복개천 | 지형 피처 추가 |
| **훈련 데이터 내 침수 부재** | 노드별 `flood_flag=1` 비율 확인 | 오버샘플링 |
| **배수구역 경계 노드** | 두 배수구역에 걸친 위치 | 양쪽 연결 보완 |

## 4. 위험 탐지 성능 평가 (분류)

### 4-1. 분류 지표 비교

| 지표 | 수식 | 의미 | 침수 예측 적합성 |
|------|------|------|----------------|
| **Precision** | TP/(TP+FP) | 경보 중 진짜 침수 비율 | ✅ 허위경보 억제 |
| **Recall** | TP/(TP+FN) | 실제 침수 중 탐지 비율 | ✅✅ **인명·재산 보호 최우선** |
| **F1** | 2·P·R/(P+R) | Precision-Recall 조화 평균 | ✅ 불균형 종합 |
| **AUROC** | TPR vs FPR 면적 | 임계값 무관 랭킹 능력 | ⚠️ **불균형에서 낙관적 과평가** |
| **AUPRC** | Precision vs Recall 면적 | 양성 클래스 탐지 집중 | ✅✅ **불균형 데이터 핵심** |

### 4-2. AUROC만으로 부족한 이유 (중요)

이 데이터의 클래스 비율: **양성(침수) : 음성(정상) = 1 : 20**

```
[ 극단적 예시: 모든 예측을 "정상"으로 내는 trivial 모델 ]
  Accuracy = 95.2%  → 무의미
  AUROC    = 0.50   → 랜덤 수준임이 드러남
  AUPRC    = 0.048  → 베이스라인(=양성 비율)이므로 낮게 드러남
  Recall   = 0.00   → 명백히 드러남

[ 잘못 좋아 보이는 모델의 예 ]
  AUROC = 0.92이지만 Recall = 0.30
  → 실제 침수 이벤트의 70%를 탐지 못하는 경보 시스템 실패 모델
```

**AUROC**: 무작위로 양성/음성 한 쌍을 뽑았을 때 양성의 점수가 더 높을 확률.  
불균형 데이터에서 음성 클래스가 압도적으로 많으면 TN이 폭발적으로 커지며 FPR이 과도하게 낮아 보여 ROC 곡선 자체가 낙관적으로 그려집니다.

**AUPRC**: Precision-Recall 곡선 면적. 양성 클래스에만 집중하여 평가하므로 불균형 데이터에서 실질적인 탐지 능력을 측정합니다. 베이스라인(무작위 분류기)의 AUPRC = 양성 비율 = 0.048이므로 이를 상회해야 의미 있는 모델입니다.

### 4-3. 분류 임계값 전략

```
기본 임계값 (0.5)가 최적이 아닐 수 있습니다.

홍수 예측 운영 원칙: Miss(탐지 실패) > False Alarm(허위경보)
→ 임계값을 0.3~0.4로 낮춰 Recall 우선

권장 방법:
  1. Precision-Recall 곡선 그리기
  2. F1-score 최대화 임계값 선택 (기본)
  3. 또는 Recall ≥ 0.80 보장 임계값 선택 (운영 우선)
  4. 선택한 임계값을 논문에 명시
```

## 5. 경보 운영 관점 평가

### 5-1. 핵심 운영 지표 정의

```
이벤트 타임라인:
  T_onset  : 수위가 경계수위(flood_flag=1) 최초 초과 시점
  T_alert  : 모델이 최초 양성 예측을 낸 시점

Detection Lead Time  = T_onset - T_alert  [분]  (양수 = 사전 경보 성공)
Detection Delay      = T_alert - T_onset  [분]  (양수 = 경보 지연)
False Alarm Rate     = FP / (FP + TN)           (정상인데 경보)
Miss Rate            = FN / (FN + TP) = 1 - Recall
Event-wise Recall    = 탐지 성공 이벤트 수 / 전체 이벤트 수
                       ※ 1개 이벤트 = 연속 침수 구간 1회 (timestep 아님)
```

### 5-2. Lead Time 평가 방법

```python
for event in flood_events:
    T_onset = event['start_timestep']
    lead_time = None
    for t in range(T_onset - max_lookback, T_onset):
        if pred_prob[t] >= threshold:
            lead_time = (T_onset - t) * 10  # 분 단위 (10분 해상도)
            break

    # 분류 기준
    if lead_time is None:
        detection_delay = ...     # 사후 탐지
    elif lead_time >= 20:
        success = True            # 대피 가능
    elif lead_time >= 0:
        partial_success = True    # 참고 수준
```

### 5-3. 운영 기준 목표값

| 지표 | 목표 기준 | 근거 |
|------|---------|------|
| Detection Lead Time | ≥ 20분 | 대피·도로 차단 조치 최소 소요 시간 |
| Event-wise Recall | ≥ 0.85 | 이벤트 단위로 85% 이상 사전 탐지 |
| False Alarm Rate | ≤ 0.10 | 월 3회 이하 허위경보 (신뢰성 유지) |
| Miss Rate | ≤ 0.15 | 이벤트 15% 이하 누락 허용 |

## 6. 이벤트 구간별 평가

### 구간 정의 및 중점 지표

| 구간 | 정의 조건 | 중점 지표 | 판단 기준 |
|------|---------|---------|---------|
| **평상시** | `flood_flag=0`, 강우 없음 | Specificity, FAR | 허위경보 억제 |
| **강우 시작** | 첫 강우 ~ 수위 반응 전 | Lead Time, Recall | 얼마나 일찍 반응하는가 |
| **수위 상승** | 수위 급증 (변화율 양수) | MAE@각 step, RMSE | 상승 속도 예측 정확도 |
| **위험수위 근접** | 경계수위 80~100% 구간 | Recall, F1 | 놓치면 피해 직결 |
| **첨두수위** | 최고 수위 ±1~2스텝 | Peak Error = `\|y_peak - ŷ_peak\|` | 극한 상황 과소/과대 추정 |
| **수위 하강** | 첨두 이후 감소 구간 | MAE, R² | 도로 통제 해제 시점 예측 |

```
구간 레이블링 예시:
  phase = np.where(level == 0, 'normal',
          np.where(level > 0.8 * max_level, 'near_peak',
          np.where(diff > 0.1, 'rising',
          np.where(diff < -0.1, 'recession', 'onset'))))
```

### 7-3. 복합 평가 체계의 필요성

도시 침수 예측 모델은 수위라는 연속값을 정확히 추정하는 **회귀 정밀도**, 위험 여부를 조기에 판별하는 **분류 민감도**, 센서 네트워크 전반에 걸쳐 공간적으로 균형 잡힌 **공간 일반화 능력**, 그리고 실제 운영 환경에서 충분한 사전 경보를 제공하는 **시간적 선행성**이라는 네 가지 요건을 동시에 충족해야 한다. MAE나 RMSE 단독으로는 위험 이벤트의 조기 탐지 능력을 전혀 반영하지 못하며, AUROC 단독으로는 1:20에 달하는 극심한 클래스 불균형 속에서 실질적인 탐지 성능을 은폐할 수 있다. 또한 전체 평균 지표는 일부 취약 노드에 집중된 공간적 오차 편향을 가릴 수 있고, 단일 예측 horizon에 대한 평가는 예측 거리에 따른 오차 감쇠 특성과 운영 유효 시간을 드러내지 못한다. 따라서 시계열 GNN 기반 도시 침수 예측 모델의 성능은 회귀·분류·공간·운영의 네 차원을 아우르는 복합 평가 체계 하에서만 그 신뢰성과 현장 적용 가능성을 종합적으로 검증할 수 있다.

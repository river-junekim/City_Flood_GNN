## 0. 과업 그래프 구조와 데이터의 관계 (요청 근거의 뼈대)
과업 = **노드-링크 상관관계 분석 모듈** (테스트베드 관악):
- **노드 = 맨홀**(관악 12,184, GIS 보유) — 관측은 수위계가 붙은 맨홀만(현재 하수 13)
- **링크 = 관로**(상류→하류 유향, 관저고 기반 흐름방향, GIS 보유)
- **위험기상 예측정보 노드** = 강우(예측)를 맨홀별 유입으로 구조화해 그래프에 입력
- **분석 모듈** = KICT 개발 알고리즘 기반 네트워크에서 노드(맨홀수위)-링크(관로흐름) 간 상관관계 산출

**골격은 개념이 아니라 이미 구축·검증됨**: KICT 관악 관망 GIS의 공식 연결(시작맨홀→끝맨홀)로 **노드 13,272·링크 16,059** 그래프 생성 + 공식 접속관수로 검증(차수 정확 80%·±1 91%·공간정합 77%) + 공식경사·소유역(집수단위 40) 부착. **비어있는 건 노드 입력(위험기상 예측 × 집수면적)과 관측 밀도뿐.**

→ **빈 곳은 ②위험기상 예측정보(노드 입력) ③집수면적(노드 입력) ④관측 노드(신설 센서); ①침수 라벨은 후속 *예측* 확장·모듈 검증용(선결조건 아님).** (그림: `09_roadmap.png`(구성요소)·`10_manhole_graph.png`(실제 그래프))

## 침수 판단 기준 (정의·출처)
본 분석의 **침수 양성 = 도로수위계 침수심 ≥ 15cm** 로 정의한다. 이는 임의 임계가 아니라 **서울시 침수 예보 발령 기준**을 따른 것이다.
- **출처**: 서울시 (news.seoul.go.kr/env/archives/522983) — *"①시간당 강우량 55mm 초과 ②15분당 강우량 20mm 초과 **③도로수위계 기준 침수심 15cm 초과** 중 어느 하나라도 해당되면 각 자치구 단위로 '침수 예보'를 발령한다."*
- 위 세 조건 중 **③ 도로수위계 수위 항(15cm)** 을 침수 양성 정의로 채택(강우 항은 별도 입력으로 사용).
- **하수 충전율(fill_rate, 만관·위험수위)은 침수 기준이 아니라 별개 신호로 분리**한다 — 도로 표면 침수 ≠ 하수관 만관.

## 진행 경과와 미완(실패) 사유 — 관악·여름 기준
- **그래프 골격 구축·검증 완료**(노드=맨홀·링크=관로·유향) → 상관관계 분석 모듈 시제 구현·실행함.
- **그러나 노드-링크 상관/전파 신호가 약함**: 도로↔하수 결합 corr ~0.05 · 센서간 전파 부재(강우+이력 GNN 0.17 ≪ persistence 0.72) · 하수 만관 도달 0%·위험수위 0.064% · 하천역류 기각(침수 90%가 도림천 평상시).
- **원인 = 노드 입력과 관측이 비어 있음**: 위험기상 예측정보(②)·집수면적(③)이 없어 '맨홀별 유입'이 구조화되지 않음 → 상관관계가 드러날 입력 자체가 부재. 관측 노드도 13개로 극희소(④).
- 관악 침수 = 만관도 하천역류도 아닌 **국지 호우 표면류** → 지형(DEM)·격자 강우가 있어야 신호가 잡힘.

### ② 위험기상 예측정보 노드 = 격자(레이더형) 강우  〔노드 입력 · 리드타임의 본체〕
| | |
|---|---|
| 요청 | 이동하는 강우장을 담은 **격자 강수자료** — 기상청 레이더 합성(HSR 등)·KICT 보정 강수장 등 가용한 것이면 됨 |
| 명세(기능 요건) | ▸점이 아닌 **공간 격자** ▸관악을 해상하기 충분한 해상도(예: **1km 이하**) ▸시간 **5~10분급** ▸상류 포함 영역(이동 관측) ▸2024~현재 ▸좌표계/격자ID/강수단위/결측코드. **※기상청 HSR(500m·5분)은 충족 *예시*이지 필수 규격 아님** — 동등한 격자자료면 가능 |
| 명세 근거(④) | **▸공간 ≤1km**: 관악 면적 **약 29.57km²**, 소유역 **40개** → 평균 소유역 **≈0.74km²**. 1km 격자(1km²)는 소유역당 채 1셀이 안 돼 *분해 최소선*이 1km이고, 500m면 소유역당 ~3셀로 맨홀별 강우 차등이 가능. (관악 전역 = 1km 격자 ~30셀 / 500m ~120셀) **▸시간 5~10분**: 도시 소유역(평균 0.74km²·불투수 높음)의 도달시간이 분~십수 분이라 강우-유출 반응이 빨라, 10분보다 성긴 자료는 첨두를 놓침. **▸상류 포함**: 리드타임 = 강우장이 상류→관악 이동 시간. 대류 셀 이류속도 ~30~50km/h → 30분 리드타임엔 상류 **~15~25km** 영역의 강우장이 필요(관악 경계만으론 리드타임 0). |
| 사유 | **리드타임은 모델이 만드는 게 아니라 "미래를 담은 입력"에서 나옴.** 두 시계 중 **(가) 기상 시계**(비가 아직 관악에 안 옴)만 10~60분 선행 제공=레이더. 관악 **(나) 수문 시계**(유출 지연)는 ~0(강우-수위 동시반응, persistence 0.72≫GNN 0.17)이라 그래프만으론 몇 분뿐 → **레이더가 리드타임의 본체**. 점 강우는 리드타임 0. |
| 리드타임 근거(③·조원로 기준) | **조원로 5-6에서 강우-수위가 거의 동시 반응 → 리드타임 ~0.** 이유: 조원로 일대 소유역은 작고(평균 ≈0.74km²)·불투수율이 높아 **도달시간(time of concentration)이 분 단위**라, 비가 내리면 도로 수위가 *거의 동시에* 상승함. 즉 조원로의 강우계(점 강우)는 빗방울이 계기에 닿는 순간 이미 비가 온 것이라 **수위를 앞설 수 없음**(리드타임 0). 상류에서 다가오는 강우장을 *먼저* 보는 **격자 강우(레이더)** 만이 조원로 수위를 시간적으로 앞서는 입력이 된다. |
| 한계(정직) | 관악 침수형=국지 대류 호우는 셀이 이동뿐 아니라 생성·소멸 → nowcasting이 가장 약한 유형. 리드타임 천장 보수적 10~30분(onset CSI 0.06~0.09가 기준선). |
| 구조 | 레이더 강우장 → nowcast → 예측 강우장 → ×집수면적 → 맨홀 유입 → 유향 GNN (2단 분리: 강우예측 단·침수라우팅 단 독립 검증) |
| 증거 | 보고서 §19 그림 28(관악 GNN) · §17 그림 26(onset) · 상세 `docs/LEADTIME_AND_RAIN_FORECAST.md` |

### ③ DEM (수치표고/LiDAR)  〔노드 입력: 맨홀별 집수면적〕
| | |
|---|---|
| 요청 | 관악 포함 **고해상 DEM/LiDAR** |
| 명세 | DEM 1m급(또는 가용 최고해상) · 관악 전역 · 좌표계/격자 정보 포함 |
| 사유 | **DEM은 노드-엣지(맨홀-관로) *연결*용이 아니다** — 관망 연결은 KICT GIS 관저고로 이미 확보·검증됨(차수 80%·공간정합 77%). DEM이 필요한 이유는 **노드 입력(맨홀별 유입)의 결손 두 가지**를 채우기 위함이다:<br>**① 맨홀별 집수면적** — 맨홀 유입 = 강우 × 집수면적(합리식 Q=C·i·A). 강우(②)만으론 유입 *부피*가 안 나오고, **면적이 곱해져야 맨홀별 유입량**이 된다. 공식 소유역은 **40개로, 맨홀 12,184개 대비 너무 coarse** → 맨홀 단위 정밀 집수면적은 **DEM 흐름누적(flow accumulation)·저지대 분석으로만** 도출 가능.<br>**② 표면류 경로** — 관악 침수는 **표면류형**이라 관망 *밖* 지표에서 물이 모인다. 관망 그래프엔 지표 지형이 없어, DEM의 저지대·지표 흐름방향이 "어느 맨홀 위로 물이 모이는가"를 제공한다.<br>**③ 노드 변별력** — 집수면적이 없으면 강우가 모든 맨홀에 *균일*하게 입력되어 노드 간 차이가 사라지고 상관 신호가 뜨지 않는다. 같은 비라도 집수면적이 큰 맨홀이 먼저 위험수위·월류에 도달하므로, 공간 위험을 구분하려면 면적 가중 입력이 필수다. |
| 비고 | **하수관망 수리모델(SWMM)은 현 단계 배제** — 초기부터 모델 복잡도가 커지므로 사용하지 않음. (필요 시 후속 단계에서 재검토) |
| 증거 | `req2_dem_rationale.png`(관악 침수=표면류) |

## C. 데이터 외 — 방향 정렬 확인 (회신 요청)
1. **KICT 개발 알고리즘**(네트워크 설계 근거) 명세·문서 공유 — 우리 맨홀-노드 유향그래프와의 정합 확인.
2. **위험기상 예측정보**의 정의·제공 형식(레이더 격자 / 초단기예측 / 특보 중 무엇인지).
3. KICT ground-truth 침수 검증 보유 여부(후속 *예측* 확장·모듈 검증용 — 이미 보유 시 ① 해소).
4. 관악 신설 29센서 제공 가능 여부·형식 · 고해상 DEM/관망 수리모델(SWMM) 보유 여부.

> 핵심: 관망·맨홀(노드-링크 골격)은 확보·검증됨. **②위험기상 예측정보·③집수면적·④관측 밀도가 상관관계 분석 모듈을 완성**하고, ①침수 라벨은 이후 *예측* 확장의 정답지다.
